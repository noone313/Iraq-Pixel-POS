## 🌟 نظرة عامة (Overview)
**Pixel POS** هو نظام سحابي متطور لإدارة نقاط البيع والمخازن، مصمم خصيصاً للشركات التي تبحث عن السرعة، الأمان، والدقة المحاسبية. يتميز النظام بواجهة مستخدم مستوحاة من فلسفة تصميم **"عراق بكسل"**، حيث تلتقي البساطة بالاحترافية العالية.

---

## ✨ المميزات الرئيسية (Core Features)

### 📊 مركز التقارير الذكي (Smart Reporting)
* **تحليل المبيعات:** رسوم بيانية تفاعلية توضح أداء المبيعات والأرباح بشكل أسبوعي وشهري.
* **توزيع المبيعات:** مراقبة نسبة المبيعات النقدية مقابل الآجلة (Cash vs Debt).
* **تصدير البيانات:** إمكانية استخراج التقارير بصيغة PDF و Excel بضغطة زر.

### 📦 إدارة المخازن (Inventory Management)
* **نظام التنبيهات:** إشعارات ذكية عند وصول المنتجات لـ "حد الطلب" (Low Stock Alerts).
* **تتبع الحركات:** سجل كامل لحركات الصادر والوارد (Stock Movements).
* **الجرد الآلي:** حساب قيمة المخزون الحالية بناءً على سعر التكلفة الفعلي.

### 💰 المحاسبة والديون (Accounting Logic)
* **توزيع الديون (FIFO):** منطق برمجي متطور يقوم بتوزيع مبالغ التسديد على الفواتير الأقدم تلقائياً.
* **كشوفات الحساب:** كشف حساب تفصيلي لكل عميل ومورد يوضح الدائن والمدين بكل دقة.

### 🌓 تجربة المستخدم (UX/UI)
* **الوضع الليلي والنهاري:** تحويل سلس وفائق النعومة (Ultra-Smooth Transition) يحفظ راحة العين.
* **واجهة بدون سكرول بار:** تصميم نظيف (Clean Design) يحاكي تطبيقات الجوال الحديثة.
* **أيقونات Lucide:** استخدام مكتبة أيقونات عصرية وسريعة التحميل.

## 🛠 التقنيات المستخدمة (Tech Stack)

| التقنية | الاستخدام |
| :--- | :--- |
| **Node.js & Express** | المحرك البرمجي للسيرفر (Back-end) |
| **PostgreSQL** | قاعدة بيانات علائقية قوية وموثوقة |
| **Sequelize (ORM)** | إدارة قاعدة البيانات وعلاقاتها البرمجية |
| **EJS** | محرك القوالب لتصميم الواجهات التفاعلية |
| **Tailwind CSS** | التنسيق العصري وتصميم الواجهات المرنة |
| **Chart.js** | عرض الرسوم البيانية والإحصائيات التفاعلية |
| **JWT & Cookies** | نظام الحماية وتشفير الجلسات (Authentication) |

## 🏗 بنية المشروع (Project Structure)

```bash
├── controllers/      # المنطق البرمجي لكل جزء (Sales, Reports, Customers)
├── models/           # تعريف الجداول والعلاقات في قاعدة البيانات (Sequelize)
├── routes/           # مسارات التطبيق وتوجيه الطلبات (Routing)
├── views/            # ملفات الواجهات (EJS Templates)
│   ├── partials/     # الأجزاء المتكررة (Header, Sidebar, Navbar)
│   └── reports/      # صفحات التقارير التفصيلية وبوابة البيانات
├── public/           # الملفات الثابتة (CSS, JS المخصص, الصور)
├── middlewares/      # فلاتر الحماية والتحقق من الصلاحيات (Auth)
└── app.js            # ملف التشغيل الرئيسي للمشروع (Entry Point)
